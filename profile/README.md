# UWE CS 2026

Central community hub for the **UWE Bristol Computer Science 2026 cohort**.

This organisation was created to provide a structured, collaborative space for students to share **module resources, notes, code, revision materials, and project work** throughout the degree.

## Purpose

The goal of this organisation is to keep academic resources organised in a way that is practical for Computer Science students.

Instead of scattered files across chat apps and cloud folders, each module has its own dedicated repository containing relevant materials such as:

* lecture notes
* lab work
* assignment guidance
* revision resources
* code examples
* past exercises
* useful links and references

This serves as a **single source of truth** for the cohort, giving all students one organised place to access up-to-date study materials.

For students who may be less organised, this provides a clear and consistent structure to follow, reducing duplication of effort and saving time across the class.

Rather than every student maintaining separate folders and duplicated resources, materials can be organised once in one shared location for everyone to benefit from.

## Repository Structure

Each module is maintained in its own repository.

Example structure:

```text id="vkj310"
uwe-cs-2026/
├── pop
├── foc
├── csa
├── wdd
```

This keeps content clean, searchable, and easy to contribute to.

## Collaboration

This organisation is designed so that **any student can collaborate and contribute**.

Students are encouraged to contribute by adding:

* personal notes
* useful study resources
* code snippets

By using Git and GitHub, students can **pull the latest changes to receive instant updates** whenever new materials, notes, or code are added.

This ensures everyone stays aligned with the most recent study resources without needing files to be manually resent through chat apps or cloud folders.

Please keep contributions clear, well-organised, and relevant to the module.

## File Organisation Guidelines

Recommended folder structure inside each module repository:

## Important Note

This organisation is intended to support collaborative learning for the class.

Please do not upload copyrighted university materials that are not permitted to be redistributed.

Use this space primarily for **student-created notes, summaries, code, and study aids**.

## Data Redundancy and Reliability

Unlike relying solely on a cloud folder, this workflow provides **built-in redundancy**.

When students clone a repository and regularly pull updates, the same study materials are stored across multiple devices rather than existing in only one location.

This means the organisation does not rely on a single Google Drive, account, or administrator to preserve important materials.

Every student who has cloned the repository effectively holds a synchronised copy of the shared resources on their own computer.

This greatly reduces the risk of accidental data loss caused by:

* deleted cloud folders
* account access issues
* accidental overwrites
* missing permissions
* sync failures

In effect, the cohort benefits from a distributed backup model where important study resources are naturally replicated across multiple machines.

Google Drive can still be used for large files such as lecture recordings or video resources, while GitHub remains the primary structured and version-controlled source of truth as we can just add a link to the Google Drive here.

## Long-Term Goal

The aim is to build a shared knowledge base that benefits the entire cohort throughout the degree and remains a useful reference beyond graduation.
